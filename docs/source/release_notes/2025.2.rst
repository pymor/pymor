pyMOR 2025.2 (January 9, 2025)
--------------------------------

We are proud to announce the release of pyMOR 2025.2!

ADD SUMMARY OF CHANGES

pyMOR 2025.1 contains contributions by Max Bindhak, Timo Plath, Sven Ullmann and
@franciuprise. See `here <https://github.com/pymor/pymor/blob/main/AUTHORS.md>`__
for more details.


Main new features
^^^^^^^^^^^^^^^^^

ML-based reductors and VKOGA algorithm
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- `[#2484] VKOGA implementation <https://github.com/pymor/pymor/pull/2484>`_
- `[#2498] Data driven reductor <https://github.com/pymor/pymor/pull/2498>`_

RB methods for saddle-point problems (Stokes equation)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
(also mention StationaryLSRBReductor)
- `[#2448] Reductor for the stationary Stokes equation <https://github.com/pymor/pymor/pull/2448>`_
- `[#2485] add saddle point model <https://github.com/pymor/pymor/pull/2485>`_

Q-DEIM algorithm
~~~~~~~~~~~~~~~~
- `[#2493] Implement Q-DEIM algorithm <https://github.com/pymor/pymor/pull/2493>`_

Bindings for FEniCSx
~~~~~~~~~~~~~~~~~~~~
- `[#2491] FEniCSx bindings <https://github.com/pymor/pymor/pull/2491>`_


Backward incompatible changes
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

`solver_options` replaced by |Solver| objects
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
- `[#2440] Replace solver_options by Solver classes <https://github.com/pymor/pymor/pull/2440>`_

Dropped Python 3.9 support
~~~~~~~~~~~~~~~~~~~~~~~~~~
- `[#2467] Increase minimum required python version to 3.10 <https://github.com/pymor/pymor/pull/2467>`_
